Free Software PDF Readers

Guía para PDF Readers "Sprints"

Equipo

Necesario

- Ordenadores preparados para navegar por Internet - ono por participantes, para encontrar anuncios

- Impresora compatible con GNU/Linux (que pueda imprimir sobres) - mira si necesitas cartas que requieran que el sobre sea la página 1 o si necesitas que las cartas y sobres se imprimas de forma separada para imprimir fácilmente

- Tinta suficiente para imprimir una carta y un sobre por cada 'bug'

- Papel

- Sobres

- Refrescos y agua para los participantes

- Acceso a aquellas cuentas de correo o listas de correo que reciben las denuncias de los bugs

- Script operativo para procesar informes de emails generados vía web (disponible en SVN)

- Plantilla de carta en formato ODT adaptada a los contactos e idioma locales

- Petición de gasto aprovada para cubrir los gastos del correo

A ser posible

- Proyector - para mostrar ejemplos de anuncios, mostrar la página de bus y proyectar identi.ca / chat room feed

- Altavoces - Para música de fondo

- Un pequeño premio para la persona que encuentre el mayor número de anuncios (por ejemplo, una caja de chocolates)

- Panfletos de FSFE leafletsy otros materiales promocionales

- Ayuda para transportar los materiales a donde se realice el evento

- Lista de páginas web públicas para trabajar

Actividades

Monta un grupo de activistas de Software Libre y:

1. Encuentra anuncioa de lectores propietarios de PDF en páginas web del Gobierno en tu país, comunidad autónoma....

2. Envía cartas a las instituciones responsables pidiéndoles quitar o reemplazar los anuncios

Puedes encontrar un modelo de carta en la página de la campaña de PDF Readers de FSFE, y otra en web SVN. Puedes modificar la carta para imprimirla fácilmente o para adecuarla a tu país/área.

No olvides escribir en microblog lo que haces y usar la etiqueta #pdfreaders cuando lo hagas. También puedes usar otras etiquetas como #openstandards or #opengov.

Opcional

- Publicar una nota de prensa antes del evento

- Preparar una entrevista con una radio local o podcast antes del evento

- Enviar una copia de las peticiones de firmas en tu país/área además de la carta referida al bug

- Manda un correo-e a las instituciones con una copia de la carta imprimida y la lista de firmas

- Envía cartas para hacer un seguimiento a aquellas instituciones que han sido contactadas en el pasado y no han modificado sus webs.

Nota sobre los scripts

He creado un script adictional para procesar bug mails. Está escrito en PHP y genera archivos .odt directamente. El actual script Python script es más operativo pero tiene peor documentación, pero tú puedes elegir cualquiera de ellos (ambos están en el SVN)
