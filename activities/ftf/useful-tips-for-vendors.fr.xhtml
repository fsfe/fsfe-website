1 Repères de conformité pour les fournisseurs - FSFE Juridique

Juridique

Repère de conformité aux licences pour les fournisseurs

Ces points de repère ont pour but de vous aider à répondre aux principales questions qui peuvent surgir lorsque vous distribuez un logiciel qui contient du code placé sous la licence GNU GPL. Pour consulter un guide complet et officiel sur la manière de diffuser un logiciel sous la licence GNU GPL, vous devriez lire la FAQ de la GNU GPL de la FSF ainsi que le didacticiel de la FSF sur l'utilisation de la licence GNU GPL.

Ce texte ne constitue pas un conseil juridique. Si vous avez des doutes, consultez un avocat.

Vérifications essentielles :

  1. Vérifiez que le code source du logiciel soit effectivement disponible pour les utilisateurs.
  2. Vérifiez que vous êtes bien le détenteur des droits d'auteur du logiciel, et si vous utilisez du Logiciel Libre appartenant à des tiers, assurez-vous de mentionner correctement leurs droits respectifs.
  3. Vérifiez que vous distribuez bien la bonne version du code source. La version du code source que vous publiez doit être exactement la même que celle du code exécutable ou code objet que vous distribuez.
  4. Vérifiez que vous incluez le code source de tout produit dérivé du code sous GNU GPL que vous utilisez, et pas uniquement le code source sous GNU GPL des composants de tierces parties.
  5. Vérifiez que vous avez bien inclus les scripts utilisés pour piloter la compilation et l'installation.
  6. Si vous publiez les outils de compilation, et que certains d'entre eux sont sous licence GNU GPL (comme le compilateur GCC), vérifiez que le code source de ces outils est également disponible.

Avant de distribuer le produit :

  1. Vérifiez que vos contrats d'achat exigent des fournisseurs qu'ils vous informent de la présence de tout logiciel sous licence GNU GPL présent.
  2. Vérifiez que vos fournisseurs mettent à disposition tout ce dont vous avez besoin pour vous conformer à la licence GNU GPL.
  3. Il est utile de vous assurer qu'en cas de manquement à la GPL, votre fournisseur accepte de régulariser cette situation dans les meilleurs délais.

À la distribution du produit :

  1. Vérifiez qu'une copie de la licence GNU GPL est fournie avec le produit.
  2. Vérifiez qu'une copie du code source est fournie avec le produit, ou que vous incluez une offre écrite de le fournir sur un support physique tel qu'un CD-ROM pour un prix ne dépassant pas les frais de production et d'expédition.
  3. Souvenez-vous que vous ne pouvez pas vous contenter d'offrir un lien pour télécharger le code source à la place de l'offre écrite d'envoi de ce code sur un médium physique. Vous pouvez toutefois ajouter à cette offre la possibilité de télécharger le code source sur un site internet, ce qui vous permettra de réduire le nombre de demandes du code source.
  4. Souvenez-vous que si vous distribuez des exécutables via Internet, vous devez alors héberger le code source sur un serveur que vous administrez ou que ce serveur est administré par quelqu'un avec qui vous avez un accord explicite.

N'oubliez pas de consulter le didacticiel de la FSF sur l'utilisation de la licence GNU GPL. Il est court, clair et explique exactement ce que vous avez à faire lorsque vous souhaitez distribuer du code sous la licence GNU GPL.
