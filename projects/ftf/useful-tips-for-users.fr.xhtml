Points de repères pour les utilisateurs sur le respect de la licence GNU GPL

Ces points de repère peuvent vous aider à identifier les principaux problèmes liés aux logiciels qui contiennent du code sous licence GNU GPL. Pour consulter un guide complet et officiel sur la licence GNU GPL, vous devriez lire la FAQ de la GNU GPL de la FSF.

Ce texte ne constitue pas un conseil juridique. Si vous avez des doutes, consultez un avocat.

Vérifications essentielles :

  1. Vérifiez qu'une copie de la licence GNU GPL est livrée avec le logiciel.
  2. Vérifiez qu'une copie du code source est livrée avec le logiciel, ou qu'il y a une proposition écrite de vous fournir le code sur un support physique, comme un CD-ROM, pour un prix ne dépassant pas les frais de production et d'expédition.
  3. Vérifiez que le distributeur ne prétend pas être le seul détenteur des droits d'auteur alors que le logiciel contient du code créé par d'autres.

Vérifications détaillées :

  1. Vérifiez que la version du code source disponible est la bonne. La version du code source fourni doit être exactement la même que celle du code exécutable ou code objet qui vous a été fourni.
  2. Vérifiez que le code source de tout produit dérivé d'un code d'une tierce partie sous GNU GPL est fourni, et pas uniquement le code source sous GNU GPL des composants de cette tierce partie.
  3. Vérifiez que le code source inclut bien les scripts utilisés pour piloter la compilation et l'installation.
  4. Si les outils de compilation sont distribués et que certains d'entre eux sont sous licence GNU GPL (comme le compilateur GCC), vérifiez que le code source de ces outils est également disponible.

S'il vous semble que le logiciel que vous avez en main ne respecte pas les termes de la licence GNU GPL, relisez la FAQ de la GNU GPL de la FSF ainsi que le didacticiel de la FSF sur l'utilisation de la GNU GPL. Si vous soupçonnez qu'il y a une violation de la licence GNU GPL vous pouvez prendre contact avec la FTF afin que nous vous aidions à résoudre le problème.
