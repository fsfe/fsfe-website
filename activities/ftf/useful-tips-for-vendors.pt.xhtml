Dicas úteis sobre conformidade para os vendedores

Essas dicas úteis podem ajudá-lo a localizar alguns problemas comuns quando lança um software que contém o código sob a licença GNU GPL. Para uma guia completa e autorizada para a liberação de software sob a GNU GPL, você deve ler as Perguntas mais frequentes sobre a GNU GPL da FSF e o HOW-TO para usar a GNU GPL da FSF .

Isto não é um aconselhamento jurídico. Se você tiver dúvidas consulte um advogado.

Diretrizes básicas

  1. Verifique se o código-fonte do software está disponível para os usuários.
  2. Verificar se você for o detentor do copyright do software e se você inclui Software Livre com copyright de terceiros, certifica que vai dar o devido crédito.
  3. Verifique se você está distribuindo a versão correta do código-fonte. O código fonte enviado deve ser o mesmo código fonte utilizado para a construção do binário.
  4. Verifique se você inclui o código-fonte de quaisquer trabalhos derivados do código GNU GPL que você está usando, não só dos componentes GNU GPL de terceiros.
  5. Verifique se você inclui os scripts utilizados para controlar a compilação e instalação.
  6. Se o conjunto de ferramentas é lançado e contém ferramentas sob a GNU GPL (como o compilador GCC), verificar se o código fonte dessas ferramentas também é liberado.

Antes de entregar o produto

  1. Verifique se os seus contratos de compra exigem dos fornecedores revelar a presença de qualquer software GPL.
  2. Verifique se os seus fornecedores oferecem todos os materiais que você precisa para cumprir a licença GNU GPL.
  3. Isso ajuda a garantir que, se o fornecedor acaba por não cumprir com a GPL, ele concorda em regularizar a sua situação em tempo hábil.

Entrega do produto

  1. Verifique se uma cópia da licença GNU GPL é fornecida com o produto.
  2. Verifique se uma cópia do código-fonte é fornecido com o produto ou que você inclui uma oferta por escrito para fornecer o código-fonte em uma mídia física, como um CD-ROM por não mais do que o custo de produção e transporte.
  3. Lembre-se que você não pode oferecer uma ligação para baixar o código fonte em vez de ofertar por escrito o envio do código-fonte em mídia física. Você pode complementar a oferta por escrito, com uma referência a um sítio web de descarga. Isto pode reduzir o número de pedidos de código-fonte.
  4. Lembre que se você distribui binários pela internet, deve hospedar o código-fonte em um servidor que você opere ou que seja operado por alguém que tenha um acordo explícito com você.

Não se esqueça de revisar o HOW-TO da FSF para usar a GNU GPL. É curto, claro e explica exatamente o que você precisa fazer quando libera o código sob a licença GNU GPL.
