FTF: Nützliche Tips zur Einhaltung der Lizenz für Anbieter

Dieser Überblick hilft Ihnen, die häufigsten Probleme beim Veröffentlichen von Software, die GNU GPL lizensierten Code enthält, zu vermeiden. Um eine komplette und verlässliche Anleitung zu erhalten, sollten Sie die FSF GNU GPL FAQ und das FSF-HOWTO zur Nutzung der GNU GPL lesen.

Diese Dokumente stellen keine Rechtsberatung dar. Konsultieren Sie im Zweifelsfall einen Anwalt.

In aller Kürze:
  1. Stellen Sie sicher, dass der Quellcode der Software für den Anwender zugänglich ist.
  2. Stellen Sie sicher, dass das Urheberrecht der Software bei Ihnen liegt und Sie den Rechten Dritter, deren Software Sie ggf. in ihrem Produkt verwenden, Rechnung tragen.
  3. Stellen Sie sicher, dass Sie die korrekte Version des Quellcodes veröffentlichen. Der mitgelieferte Quellcode muss mit demjenigen identisch sein, aus dem das fertige Programm erstellt wurde.
  4. Stellen Sie sicher, dass Sie den vollständigen Quellcode aller verwendeten (Hilfs-)Programme beigefügt haben, und nicht etwa nur derjenigen Programme oder Programmteile, die selbst schon GNU GPL lizensiert sind.
  5. Stellen Sie sicher, dass das Paket die zum Kompilieren und Installieren notwendigen Skripte enthält.
  6. Falls Sie ein Programm veröffentlichen, das Werkzeuge, die unter der GNU GPL lizensiert wurden, verwendet (z.B. den GCC Compiler), so stellen Sie sicher, dass auch der Quellcode dieser Werkzeuge für den Anwender zugänglich ist.
Vor dem Verteilen der Software:
  1. Überprüfen Sie, ob Ihre Kaufverträge Sie verpflichten, Ihren Lieferanten die Verwendung von GPL-lizensierter Software mitzuteilen.
  2. Überprüfen Sie, ob ihre Lieferanten ihnen alle Materialien die Sie zur Erfüllung der GNU GPL Lizenz brauchen zur Verfügung stellen.
  3. Es hilft sicher zu stellen, dass wenn ihr Lieferant nicht der GPL entspricht, dieser zustimmt die Situation in einer zeitgerechten Art und Weise zu verbessern.
Das Verteilen der Software:
  1. Stellen Sie sicher, dass Ihrem Produkt eine Kopie der GNU GPL beiliegt.
  2. Stellen Sie sicher, dass entweder der Quellcode der Software mit ausgeliefert wird, oder das schriftliche Angebot, den Quellcode auf einem physischen Medium (etwa einer CD-ROM) auf Verlangen für nicht mehr als die Kosten der Produktion und dem Versand zur Verfügung zu stellen.
  3. Beachten Sie, dass es nicht ausreicht einen Downloadlink, anstelle des schriftlichen Angebotes den Quellcode auf einem physichen Medium nachzuliefern, anzubieten. Sie können jedoch Ihr Angebot, den Quellcode nachzuliefern, um den Link zu einer Downloadseite ergänzen, wodurch Sie die Häufigkeit des Versendens des Quellcode wahrscheinlich verringern können.
  4. Wenn Sie Binärdateien über das Internet verbreiten, beachten Sie, dass der Quellcode auf einem Server verfügbar sein muss, der entweder von Ihnen betrieben wird oder von jemandem, mit dem Sie ein ausdrückliches Abkommen haben.

Vergessen sie nicht sich das FSF-HOWTO zur Nutzung der GNU GPL anzusehen. Es ist kurz, verständlich und erklärt ihnen genau wie Sie vorgehen müssen, wenn Sie Code unter der GNU GPL veröffentlichen.
