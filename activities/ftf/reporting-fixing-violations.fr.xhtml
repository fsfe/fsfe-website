1 Signaler et résoudre les violatoins de licence - FSFE Juridique

Juridique

Signaler et résoudre les violations de licence

Ce guide vous présente quelques trucs pratiques pour résoudre des problèmes communs de respect des licences de Logiciels Libres. Ce texte ne constitue pas un conseil juridique. Si vous avez des doutes, consultez un avocat qualifié.

Signaler une violation

Soyez prudent lorsque vous signalez une violation de licence. Les accusations et suspicions portées sur des listes de diffusion publiques créent du trouble mais n'avancent en rien dans la résolution du problème. En vérifiant vos accusations vous aiderez les experts à résoudre les violations rapidement.

Un signalement de violation efficace envers des entreprises à propos d'un produit potentiellement délictueux devrait contenir :

  • Le nom du produit affecté
  • La raison pour laquelle la violation est supposée constituée
  • Le nom du projet dont le code est supposé avoir été violé
  • Une déclaration indiquant sous quelle licence est publié ce code
  • Un lien vers le site du projet

Un signalement de violation envers une organisation comme gpl-violations.org ou la FTF devrait contenir :

  • Le nom du projet dont le code est supposé avoir été violé
  • Une déclaration indiquant sous quelle licence est publié ce code
  • Un lien vers le site du projet
  • Le nom et le site web de la partie qui est supposée violer le code
  • La raison pour laquelle la violation est supposée constituée

Quelques points de repères additionnels :

  • Ne faites pas suivre de longs fils de discussion par courriel. Ils ne font que rendre plus difficile la tâche d'évaluation de la situation.
  • Si vous avez une preuve claire de la violation, c'est une bonne idée d'en informer les détenteurs des droits d'auteurs. Ils pourront entamer une action judiciaire si nécessaire.

Vous pouvez envoyer vos signalements de violations à :

  • gpl-violations.org : license-violation@gpl-violations.org
  • La Freedom Task Force de la FSFE : legal@lists.fsfe.org

Traiter un signalement de violation

Il est important de traiter avec soin tout signalement de violation. Le développement des Logiciels Libres se concentre sur les engagements de la communauté et sur une communication claire. Cela veut dire qu'il est primordial de répondre aux problèmes rapportés, même si celle-ci doit être initialement brève. Cela permet de désamorcer l'escalade.

Quelques étapes utiles :

  • Confirmez que vous avez reçu tous les rapports qui vous ont étés envoyés et informez l'expéditeur que vous examinez le dossier
  • Si le signalement a été fait sur un forum public, tentez de faire migrer la discussion vers un espace privé dès que possible
  • Isolez précisément le problème. Si vous n'avez pas encore ces informations, demandez au rapporteur de préciser :
    • Le nom du produit affecté ou le code exact qui pose problème
    • La raison pour laquelle la violation est supposée constituée
    • Le nom du projet dont le code est supposé avoir été violé
    • Une déclaration indiquant sous quelle licence est publié ce code
    • Un lien vers le site du projet
  • Tenez informé le rapporteur des nouvelles informations dès que vous les recevez

Et enfin gardez à l'esprit :

  • Tous les rapporteurs ne comprennent pas forcément complètement les licences et il peut y avoir des erreurs dans leurs rapports
  • Le respect des termes des licences n'est pas optionnel et un manque de respect peut avoir des conséquences sérieuse
  • Vous pouvez embaucher des ingénieurs dans le domaine du respect des licences ou vous assurer des services de tierces parties si nécessaire

Vous obtiendrez plus d'informations sur les bonnes pratiques en ce domaine en contactant :

  • La Freedom Task Force de la FSFE : legal@lists.fsfe.org

Vous pouvez obtenir une assistance d'ingénierie en respect des licences en contactant :

  • Tjaldur Software Governance Solutions: http://www.tjaldur.nl

Prévenir une violation

La meilleur manière de résoudre les violations est de les empêcher de se produire.

Quelques points de repères utiles :

  • Lisez les licences que vous allez utiliser
  • Informez-vous sur les sites Internet expliquant ces licences
  • Obtenez des avis d'experts

Quelques points de repères utiles pour la chaîne d'approvisionnement :

  • Si des tierces parties vous fournissent du code, assurez-vous que le respect des licences est stipulé dans vos contrats
  • Demandez aux fournisseurs de prendre en charge le coût de résolution des violations

Pour plus d'informations, vous pouvez contacter :

  • gpl-violations.org : legal@lists.gpl-violations.org
  • La Freedom Task Force de la FSFE : legal@lists.fsfe.org
