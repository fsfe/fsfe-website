Estes conselhos úteis podem ajudar a localizar algumas questões comuns quando é lançado um software que contém código sob a licença GNU GPL. Para um guia completo e oficial para lançamento de software sob a licença GNU GPL deverá ler as FAQs da FSF sobre GNU GPL e os HOW-TO da FSF para usar a GNU GPL.

Isto não é um aconselhamento jurídico. Se tiver dúvidas consulte um advogado.

Linhas de orientação básicas:
  1. Verificar que o código fonte do software está disponível para os utilizadores.
  2. Verificar que você é o titular do copyright do software e que inclui os copyrights do Software Livre de terceiros, garantindo que exibe os devidos créditos.
  3. Verificar que você está a distribuir a versão correcta do código fonte. O código fonte distribuido deve ser o mesmo código que foi utilizado para construir o binário.
  4. Verificar que você inclui o código fonte de quaisquer trabalhos derivados do código GNU GPL que estiver a utilizar, não apenas os componentes GNU GPL de terceiros.
  5. Verificar que você inclui os scripts usados para controlar a compilação e a instalação.
  6. Se o conjunto de ferramentas for lançado e contiver ferramentas sob a licença GNU GPL (como o compilador GCC), verificar que o código fonte dessas ferramentas também é disponibilizado.
Antes de enviar o producto:
  1. Verificar que os seus contratos de compra requerem que os fornecedores informem da existência de qualquer software GPL.
  2. Verifique que os seus fornecedores disponibilizem todos os materais que necessita para agir de acordo com a licença GNU GPL.
  3. Ajuda assegurar-se de que se os seus fornecedores não agirem de acordo com a GPL, eles concordem em rectificar a situação em tempo útil.
Enviar o producto:
  1. Verificar que uma cópia da licença GNU GPL é enviada juntamente com o producto.
  2. Verificar que uma cópia do código fonte é enviada juntamente com o producto ou que você inclui uma proposta escrita para providenciar o código fonte num suporte físico tal como num CD ROM, cujo custo não exceda o da sua produção e envio.
  3. Lembre-se que não pode oferecer um link para download do codigo fonte no lugar de uma proposta escrita de envio do suporte físico. Pode complementar a sua proposta escrita com uma referência a um site para download. Isto pode reduzir a quantidade de pedidos do código fonte.
  4. Lembre-se que em caso de distribuir os binários na Internet então também deverá disponibilizar o código fonte nesse mesmo servidor.

Não se esqueça de rever os HOW-TO da FSF para usar a GNU GPL. É curto, claro e explica exactamente o que necessita de fazer quando distribuir código sob a licença GNU GPL.
