Lectores de PDF de Software Libre

Guía para "Sprints" sobre Lectores de PDF

Equipo

Necesario

  1. Ordenadores preparados para navegar por Internet - uno por participante, para encontrar anuncios
  2. Impresora compatible con GNU/Linux (que pueda imprimir sobres) - mira si necesitas cartas que requieran que el sobre sea la página 1 o si necesitas que las cartas y sobres se impriman de forma separada para imprimir fácilmente
  3. Tinta suficiente para imprimir una carta y un sobre por cada fallo
  4. Papel
  5. Sobres
  6. Refrescos y agua para los participantes
  7. Acceso a aquellas cuentas de correo o listas de correo que reciben las denuncias de los fallos
  8. Script operativo para procesar informes de correos electrónicos generados vía web (disponible en SVN)
  9. Plantilla de carta en formato ODT adaptada a los contactos e idioma locales
  10. Petición de gasto aprobada para cubrir los gastos del correo

A ser posible

  1. Proyector - para mostrar ejemplos de anuncios, mostrar la página de fallos y proyectar microblogueo o la sala de chat
  2. Altavoces - Para música de fondo
  3. Un pequeño premio para la persona que encuentre el mayor número de anuncios (por ejemplo, una caja de bombones)
  4. Panfletos de la FSFE y otros materiales promocionales
  5. Ayuda para transportar los materiales a donde se realice el acto
  6. Lista de páginas web públicas para trabajar

Actividades

Monta un grupo de activistas de Software Libre y:

  1. Encuentra anuncios de lectores propietarios de PDF en páginas web del Gobierno en tu país, comunidad autónoma....
  2. Envía cartas a las instituciones responsables pidiéndoles quitar o reemplazar los anuncios
  3. Puedes encontrar un modelo de carta en la página de la campaña de Lectores de PDF de la FSFE, y otra en la web SVN. Puedes modificar la carta para imprimirla fácilmente o para adecuarla a tu país/área.

No olvides escribir en microblog lo que haces y usar la etiqueta #pdfreaders cuando lo hagas. También puedes usar otras etiquetas como #openstandards or #opengov.

Opcional

  1. Publicar una nota de prensa antes del acto
  2. Preparar una entrevista con una radio local o podcast antes del acto
  3. Enviar una copia de las peticiones de firmas en tu país/área además de la carta referida al fallo
  4. Manda un correo-e a las instituciones con una copia de la carta imprimida y la lista de firmas
  5. Envía cartas para hacer un seguimiento a aquellas instituciones que han sido contactadas en el pasado y no han modificado sus webs.

Nota sobre los scripts

Hay un script adictional para procesar corresos sobre fallos. Está escrito en PHP y genera archivos .odt directamente. Echa un vistazo al archivo "readme" en el script para más información. Este script está diseñado para ser usado y puesto en marcha fácilmente (envía sugerencias o comentarios a la lista de correo web@fsfeurope.org).
