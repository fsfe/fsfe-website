Este guia rápido pode ajudá-lo a determinar alguns aspectos comuns em software que contenha código sob a licença GNU GPL. Para um guia completo e bem documentado acerca da licença GNU GPL deve ler as FAQs da FSF sobre a GNU GPL.

Isto não é um aconselhamento jurídico. Se tiver dúvidas consulte um advogado.

Uma lista de verificação rápida do utilizador

Linhas de orientação básicas:
  1. Verificar se uma cópia da licença GNU GPL veio juntamente com o software.
  2. Verificar se uma cópia do código fonte veio junto com o software ou existe uma proposta escrita de o disponibilizar num suporte físico tal como num CD ROM, cujo custo não exceda o da sua produção e envio.
  3. Verificar que o distribuidor não reinvindique em exclusivo o copyright do software se o software também contiver código criado por terceiros.
Linhas de orientação pormenorizadas:
  1. Verificar se a versão correcta do código fonte existe. O código fonte tem de ser exactamente da mesma versão que o código (binário) executável fornecido.
  2. Verificar se o código fonte inclui a fonte de quaisquer trabalhos derivados do código GNU GPL, não apenas os componentes GNU GPL de terceiros em sí.
  3. Verificar se o código fonte inclui os scripts usados para controlar a compilação e a instalação.
  4. Se o conjunto de ferramentas é disponibilizado e se contém ferramentas sob a licença GNU GPL (como o compilador GCC), verificar que o código fonte dessas ferramentas também está disponível.
  5. Se o software for distribuido pela internet, verificar que o texto da licença e o código fonte estão no mesmo servidor que o binário descarregado, a não ser que seja uma distribuição não-comercial.

Se está preocupado de que algum do software que recebeu não está conforme os termos da licença GNU GPL, pode rever as FAQs da FSF sobre a GNU GPL e HOW-TO da FSF sobre a utilização da GNU GPL. Se suspeitar da existência de uma violação da licença GNU GPL pode contactar a FTF e nós ajudaremos a resolver o assunto.
