FTF: Consigli utili per gli utenti di software

Questi consigli vi possono aiutare ad affrontare problemi comuni nel software con codice rilasciato sotto la licenza GNU GPL. Per delle informazioni dettagliate ed autorevoli riguardo la GNU GPL dovreste leggere le FAQ sulla GNU GPL della FSF.

Questo documento non costituisce un parere legale. In caso di dubbi consultate un legale di fiducia.

Linee guida essenziali:

  1. Controllate se una copia della licenza GNU GPL è rilasciata con il software.
  2. Controllate se una copia del codice sorgente è rilasciata con il software o se esiste un'offerta scritta di rilasciare il codice sorgente su un dispositivo fisico, come un CD ROM, per una spesa non superiore al costo di produzione e spedizione di tale supporto.
  3. Controllate che il distributore non dichiari di detenere il diritto d'autore esclusivo sul software nel caso questo contenga anche codice scritto da altri.

Linee guida dettagliate:

  1. Controllate se è disponibile la versione corretta del codice sorgente. Il codice sorgente deve corrispondere esattamente alla stessa versione del codice eseguibile (oggetto) fornito.
  2. Controllate se il codice sorgente distribuito è completo, ossia se include il sorgente di qualsiasi opera derivata del codice originariamente distribuito sotto GNU GPL, e non solamente il codice sorgente delle componenti GNU GPL di terze parti.
  3. Controllate se il codice sorgente include gli script per controllare la compilazione e l'installazione.
  4. Se viene distribuita anche la toolchain (gli strumenti necessari per la compilazione) e questa contiene strumenti coperti dalla GNU GPL (come il compilatore GCC), controllate se anche il codice sorgente per tali strumenti è disponibile.

Se temete che del software che avete ricevuto non rispetti i termini della GNU GPL potete controllare le FAQ sulla GNU GPL della FSF e l'HOWTO della FSF per usare la GNU GPL. Se sospettate che ci sia stata una violazione della GNU GPL potete contattare la FTF e vi aiuteremo a risolvere il problema.
