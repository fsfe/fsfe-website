Wie können Sie bei der Webseite helfen

Diese Seite versucht Freiwilligen eine Übersicht, Einführung und Referenzinformationen zur Pflege der Webseite zu geben.

Einführung

Die Techniken und Programme die zur Verwaltung der FSF Europe Webseite eingesetzt werden sollte vielen Entwicklern und Autoren bereits bekannt sein und könnte für Sie interessant sein wenn Sie sie noch nicht kennen.

Übersetzer und Freiwillige werden wahrscheinlich mit folgendem in Berührung kommen

  • Savannah -- für CVS (siehe unten) Dienste und Zugriffskontrolle
  • XHTML -- wovon die statischen Seiten alle zwei Stunden generiert werden
  • Concurrent Version System (CVS) -- zur Koordination und Synchronisation

Freiwillige die sich tiefer mit der Verwaltung der Webseite beschäftigen wollen sollten auch vertraut sein mit

  • Perl -- für die Skripte zur Verwaltung und Erstellung der Seite
  • Apache -- als Webserver

Erste Schritte

Als erstes sollten Sie sich einen Savannah Zugang einrichten, falls sie noch keinen haben. Bitten Sie einen der FSF Europe Savannah Projekt Admins Sie zum FSF Europe Projekt auf Savannah hinzuzufügen.

Sie sollten sich auch auf der Mailingliste web@fsfeurope.org eintragen, auf dieser Liste wird die Arbeit an der Webseite koordiniert.

Außerdem wird empfohlen sich auf web-cvs-notify@fsfeurope.org einzutragen, um über die Änderungen an der Seite auf dem Laufenden zu sein.

Wenn Sie zur FSF Europe Seite hinzugefügt wurden, dann lesen Sie die Anweisungen auf Savannah um die Dateien aus dem CVS zu laden an denen Sie arbeiten wollen.

Nachdem Sie den nächsten Abschnitt gelesen haben können Sie mit der Arbeit beginnen

Wichtig

Dadurch, dass die FSF Europe Webseite eine europäische/globale Ebene hat und eine, durch den Anwender wählbare, lokale Ebene, welche die Seite mit landesspezifisch Informationen erweitert, hat die Seite eine gewisse Komplexität welche dem Anwender der Seite verborgen bleibt.

[DOKUMENTATION] Wenn Sie nicht nur Seiten Übersetzen wollen (siehe unten), dann schauen Sie sich unbedingt die Referenz zur Struktur an.

[DOKUMENTATION] Es gibt auch eine Texinfo Dokumentation "Webmastering FSF Europe" von Jonas Öberg, welcher das System zur automatischen Generierung der Seiten entwickelt hat. Bitte lesen sie es um mehr über den Aufbau der Seite zu erfahren.

Schnell-Referenz für Übersetzer / Einige allgemeine Regeln

Diese einfachen Regeln sollen Sie vor Problemen und Frustration schützen und es Ihnen einfach machen Übersetzungen zu erstellen.

  • Bearbeiten Sie niemals die HTML Dateien, bearbeiten Sie stattdessen die XHTML Dateien!

  • Wenn Sie Seiten mit fehlerhaften XHTML auf den CVS Server laden, dann werden solange keine HTML Seiten erzeugt bis jemand den Fehler behoben hat. Laden Sie also niemals eine Seite auf den CVS Server bevor sie den XHTML Syntax überprüft haben. Im Verzeichnis tools des CVS befindet sich ein Skript "validate.pl", welches zur Überprüfung des Syntax verwendet werden kann.

  • Seiten werden nach folgenden Regeln benannt: "<name>.<sprache>.xhtml", <sprache> ist dabei ein aus zwei Buchstaben bestehender Ländercode. Wenn kein Ländercode existiert, dann wird davon ausgegangen, dass die Seite in englisch ist.
    (Beispiel: Das Skript bei der Generierung der Webseite geht davon aus, dass test.it.xhtml eine italienische Übersetzung der französischen Seite test.fr.xhtml ist.)

  • Übersetzen sie keine Links innerhalb der FSF Europe Webseite. Wenn es eine Link zu einer weiteren seitex.html auf der FSF Europe Webseite gibt, fügen Sie keinen Ländercode ein, dies wird bei der Erstellung der HTML Seiten automatisch angepasst.

  • Wenn Sie eine französische Übersetzung der italienischen Seite x.it.xhtml erstellen wollen, dann müssen Sie die Datei nur nach x.fr.xhtml kopieren, übersetzen und auf den CVS Server laden. Bei der nächsten Aktualisierung der Webseite wird die neue Übersetzung erkannt und in die Seite eingebunden.

  • Einige Dateien für automatisch erstellte Menüs sind im XML Format. Alles was über XHTML Seiten gesagt wurde gilt auch für diese Seiten.
