This compliance checklist can help you locate some common issues in software that contains code under the GNU GPL licence.

This is not legal advice. If you have doubts consult a legal advisor.

GNU GPL User Checklist

Basic guidelines:
  1. Make sure a copy of the GNU GPL licence comes with the software.
  2. Make sure a copy of the source code comes with the software or there is a written offer to provide the source code on a physical media like a CD ROM for no more than the cost of production and shipping.
  3. Make sure the software is not copyrighted solely by the distributor if the software contains code created by others.
Detailed guidelines:
  1. Make sure the correct version of the source code is available. The source code has to be exactly the same version as the executable (object) code provided.
  2. Make sure the source code includes the source for any derivative works of the GNU GPL code, not just of the third-party GNU GPL components themselves.
  3. Make sure the source code includes the scripts used to control compilation and installation.
  4. If the toolchain is released and contains tools under the GNU GPL (like the GCC compiler), make sure the source code for those tools is also available.
  5. If the software is distributed through the Internet the licence text and source code should be on the same server as the binary download unless it is a non-commercial distribution.
