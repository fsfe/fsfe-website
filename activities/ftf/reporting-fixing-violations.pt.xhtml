Relatar e corrigir violações de licença

Este guia apresenta algumas dicas práticas para resolver problemas comuns de conformidade com as licenças do Software Livre. Não é um parecer jurídico, e em caso de dúvida, entre em contato com um advogado qualificado.

Denunciar uma violação

Tenha cuidado ao reportar uma violação. Acusações e suspeitas expressas em listas de discussão públicas criam incerteza e pouco fazem para resolver violações. Ao verificar os fatos você pode ajudar os peritos para resolver as violações rapidamente.

Para ser útil, um comunicado de violação para as empresas e sobre um produto potencialmente infrator deve conter:

  • O nome do produto afetado
  • A razão pela qual se acredita que existe uma violação
  • O nome do código do projeto que pode ter sido violado
  • Uma declaração sobre qual licença esse código está licenciado
  • Uma ligação para o sítio web do projeto

Para ser útil, um comunicado de violação para organizações como gpl-violations.org ou a FTF deve conter:

  • O nome do código do projeto que pode ter sido violado
  • Uma declaração sobre qual licença esse código está licenciado
  • Uma ligação para o sítio web do projeto
  • O nome e o sítio web da parte que possa estar a violar o código
  • A razão pela qual se acredita que existe uma violação

Outras dicas:

  • Por favor, não transmitir tópicos longos nos correios eletrônicos. Eles fazem que seja difícil avaliar a situação.
  • Se você tem evidências claras de violação, é uma boa ideia dizer-lho aos detentores do copyright. Eles podem tomar medidas legais se necessário.

Você pode enviar comunicados de violação para:

  • gpl-violations.org: license-violation@gpl-violations.org
  • A Freedom Task Force da FSFE: legal@lists.fsfe.org

Como lidar com um comunicado de violação

É importante lidar cuidadosamente com os comunicados de violação. O desenvolvimento do Software Livre centra-se no compromisso da comunidade e uma comunicação clara. Isso significa que é importante para responder às questões apresentadas, mesmo se a sua resposta é inicialmente breve. Isso ajuda a evitar uma intensificação.

Estes são alguns passos úteis:

  • Confirme que você tem recebido os comunicados enviados e informe o informante que está investigando o caso
  • Se o comunicado foi feito em um fórum público tente mover a discussão para um espaço não-público, logo que possível
  • Isolar o problema concreto. Se você não tiver a informação, pedir ao informante:
    • O nome do produto afetado ou o código exato que causa um problema
    • A razão pela qual se acredita que existe uma violação
    • O nome do código do projeto que pode ter sido violado
    • Uma declaração sobre qual licença esse código está licenciado
    • Uma ligação para o sítio web do projeto
  • Enviar atualizações para o informante quando estiveram disponíveis

Lembre-se:

  • Nem todos os informantes entendem totalmente as licenças e pode haver erros nas suas apresentações
  • O cumprimento dos termos das licenças não é opcional e a falta de respeito pode ter consequências graves
  • Você pode contratar engenheiros ou comprar serviços de terceiros se necessário

Você pode obter mais informações sobre as melhores práticas neste campo contatando com:

  • A Freedom Task Force da FSFE: legal@lists.fsfe.org

Você pode obter suporte de engenharia entrando em contato com:

  • Tjaldur Software Governance Solutions: http://www.tjaldur.nl

Impedindo uma violação

A melhor maneira de corrigir violações é evitar que ocorram.

Dicas úteis:

  • Leia as licenças que você vai usar
  • Confira os sítios web que explicam essas licenças
  • Obter o parecer de peritos

Dicas úteis para o gerenciamento da cadeia de abastecimento:

  • Se terceiros fornecê-lo com código, certifique-se do cumprimento da licença estipulada nos seus contratos
  • Solicite dos fornecedores suportar o custo de resolver as violações

Para obter mais informações pode contatar:

  • gpl-violations.org: legal@lists.gpl-violations.org
  • A Freedom Task Force da FSFE: legal@lists.fsfe.org
