These useful tips can help you locate some common issues when releasing software that contains code under the GNU GPL licence. For a complete and authoritative guide to releasing software under the GNU GPL you should read the FSF's GNU GPL FAQ and the FSF's HOW-TO for using the GNU GPL.

Isto não é um aconselhamento jurídico. Se tiver dúvidas consulte um advogado.

Uma lista de verificação rápida do fornecedor

Orientações básicas:
  1. Verificar se o código fonte do software está disponível para os utilizadores.
  2. Verificar se você é o proprietário do copyright do software e se inclui os copyrights do Software Livre de terceiros garanta que exibe os devidos créditos.
  3. Verificar se você está a distribuir a versão correcta do código fonte. O código fonte distribuido deve ser o mesmo código utilizado para construir o binário.
  4. Verificar se você inclui o código fonte de quaisquer trabalhos derivados do código GNU GPL que estiver a utilizar, não apenas os componentes GNU GPL de terceiros em sí.
  5. Verificar se você inclui os scripts usados para controlar a compilação e a instalação.
  6. Se o conjunto de ferramentas for lançado e contiver ferramentas sob a licença GNU GPL (como o compilador GCC), verificar se o código fonte dessas ferramentas também é disponibilizado.
Antes de expedir o producto:
  1. Verificar que os seus contratos de compra requerem que os fornecedores informem da existência de qualquer software GPL.
  2. Verifique que os seus fornecedores disponibilizem todos os materais que necessita para agir de acordo com a licença GNU GPL.
  3. Ajuda assegurar-se de que se os seus fornecedores não agirem de acordo com a GPL, eles concordem em rectificar a situação em tempo útil.
Distribuir o producto:
  1. Verificar se uma cópia da licença GNU GPL é distribuida juntamente com o producto.
  2. Verificar se uma cópia do código fonte é enviada juntamente com o producto ou que você inclui uma proposta escrita de providenciar o código fonte num suporte físico tal como num CD ROM, cujo custo não exceda o da sua produção e envio.
  3. Lembre-se que não pode oferecer um link para download do codigo fonte no lugar de uma proposta escrita de envio do suporte físico. Pode complementar a sua proposta escrita com uma referência a um site para download. Isto pode reduzir a quantidade de pedidos do código fonte.
  4. Lembre-se que em caso de distribuir os binários na Internet comercialmente também deverá disponibilizar o código fonte no mesmo servidor.

Não se esqueça de rever os HOW-TO da FTF sobre a utlização da GNU GPL. É curto, claro e explica exactamente o que necessita de fazer quando distribuir código sob a licença GNU GPL.
