Deze FAQ bevat antwoorden op veelgestelde vragen van verkopers van producten met uitvoerbare code onder de GPL.

Dit is geen juridisch advies. Als u twijfels hebt, raadpleeg dan een juridisch adviseur.

Wat zijn goede praktijken voor een verkoper van op de GNU GPL gebaseerde producten?

  • Onthoud steeds dat het een verplichting is om ofwel de broncode samen met uw product te leveren, ofwel een geschreven verklaring bij het product te voegen, waarin je verklaart op eenvoudig verzoek de broncode te bezorgen. Een CD met de broncode bij het product meeleveren is een ideale manier om hieraan te voldoen.
  • Als u GPL software gebruikt, moet u een copy van de GPL-tekst bij uw documentatie voegen en duidelijk vermelden dat u gebruik maakt van GPL software.
  • Als u software of firmware updates aanbiedt via het internet, die GPL software bevatten, moet u ook de overeenkomende broncode van die update beschikbaar maken.
  • Raadpleeg onze checklist voor licenties.

Wat kan ik doen als mijn bedrijf in overtreding is met de licentie?

Breng het in orde. Dit kan op juist dezelfde manier dan voor niet-vrije softwarelicenties.

Als u de licentie overtreedt bij GNU/Linux, dan schend je het auteursrecht van ontelbaar veel partijen, waaronder bedrijven zoals Intel, IBM en HP; deze drie bedrijven hebben het auteursrecht over aanzienlijke delen van de linux kernel. Deze overtredingen worden ernstig genomen en moeten worden rechtgezet.

Waar kan ik, als verkoper, hulp vinden?

Er is zeer veel documentatie en informatie beschikbaar. Een eerste plaats waar u hulp kan vinden is bij de 'leverancier' van jouw GPL software componenten. U kan ook een advocaat raadplegen. De GPL is zo duidelijk mogelijk geschreven, maar een advocaat kan nuttig zijn voor complexe zaken zoals afgeleide werken (derivative works).

Websites zoals Savannah bieden hulp door het hosten van GPL versies van software.

Hoe maak ik de gemeenschap gelukkig?

De grootste bijdrage die u aan de gemeenschap kan leveren is toegang geven tot uw code. Software via een website voor iedereen toegankelijk maken, zoals bij Savannah, is ook zeer nuttig. Sommige verkopers proberen een gemeenschap van ontwikkelaars op te bouwen rond hun code, andere publiceren de code en kijken er niet naar om. Deze keuze is volledig vrij.

Zal het gebruik van GPL-code mij veel kosten?

Nee. Je enige verplichting is het beschikbaar stellen van de code van de GPL-componenten, ofwel een schriftelijke verklaring toevoegen dat je de code zal leveren aan niet meer dan de kost van het verzenden. Je hebt geen verplichtingen aan derden, je moet geen ondersteuning (support) geven aan anderen die jouw gewijzigde code gebruiken. De GPL verwerpt ook elke vorm van garantie.

Het toevoegen van de GPL-licentie aan de gebruikershandleiding, duidelijk vermelden dat je GPL-software gebruikt en een zip-file met de overeenkomende code op je product-CD-ROM, volstaan reeds om aan de licentie te voldoen.

Het niet vervullen van de licentievoorwaarden, is de enige situatie waarin de GPL je veel geld kan kosten.
