Nützliche Tipps zur Lizenzeinhaltung für Anbieter

Dieser Überblick soll Ihnen helfen, die häufigsten Probleme beim Veröffentlichen von Software, die GNU GPL-lizenzierten Code enthält, zu vermeiden. Um eine komplette und verlässliche Anleitung zu erhalten, sollten Sie die FSF GNU GPL FAQ und das FSF-HOWTO zur Nutzung der GNU GPL (beides auf Englisch) lesen.

Dieser Text stellt keine Rechtsberatung dar. Konsultieren Sie im Zweifelsfall einen Anwalt.

Grundlegende Richtlinien

  1. Stellen Sie sicher, dass der Quellcode der Software für den Anwender zugänglich ist.
  2. Stellen Sie sicher, dass das Urheberrecht der Software bei Ihnen liegt und Sie den Rechten Dritter, deren Software Sie gegebenenfalls in ihrem Produkt verwenden, Rechnung tragen.
  3. Stellen Sie sicher, dass Sie die korrekte Version des Quellcodes veröffentlichen. Der mitgelieferte Quellcode muss mit demjenigen identisch sein, aus dem das fertige Programm erstellt wurde.
  4. Wenn Sie GPL-lizenzierte Komponenten für Ihre Software verwenden, stellen sie sicher, dass Sie nicht nur die Original-Quellen, sondern (wenn Sie Änderungen vorgenommen haben) auch den Quellcode Ihrer Änderungen beilegen.
  5. Überprüfen Sie, ob Sie den Quellcode jeder vom GNU GPL-Code abgeleiteten Arbeit, den sie verwenden und nicht nur den der GNU GPL-Komponenten der Drittanbieter bereitstellen.
  6. Stellen Sie sicher, dass das Paket die zum Kompilieren und Installieren notwendigen Skripte enthält.
  7. Falls Sie ein Programm veröffentlichen, das GPL-lizenzierte Werkzeuge verwendet (z.B. den GCC-Compiler), so stellen Sie sicher, dass auch der Quellcode dieser Werkzeuge für den Anwender zugänglich ist.

Vor dem Verteilen der Software

  1. Überprüfen Sie, ob Ihre Kaufverträge Ihre Lieferanten dazu verpflichten, die Verwendung von GPL-lizenzierter Software offen zu legen.
  2. Überprüfen Sie, ob ihre Lieferanten alle Materialien bereitstellen, die zur Erfüllung der GNU GPL-Lizenz benötigt werden.
  3. Falls ihre Lieferanten die Bedingungen der GPL nicht erfüllen, sollten Sie sie verpflichten, das in angemessener Zeit zu berichtigen.

Das Verteilen der Software

  1. Stellen Sie sicher, dass Ihrem Produkt eine Kopie der GNU GPL beiliegt.
  2. Stellen Sie sicher, dass entweder der Quellcode der Software mit ausgeliefert wird, oder das schriftliche Angebot, den Quellcode auf einem physischen Medium (etwa einer CD-ROM) auf Verlangen für nicht mehr als die Kosten von Produktion und Versand zur Verfügung zu stellen.
  3. Beachten Sie, dass es nicht ausreicht, einen Downloadlink anstelle des schriftlichen Angebotes, den Quellcode auf einem physischen Medium nachzuliefern, anzubieten. Sie können jedoch Ihr Angebot, den Quellcode nachzuliefern, um den Link zu einer Downloadseite ergänzen, wodurch Sie die Häufigkeit des Versendens des Quellcode wahrscheinlich verringern können.
  4. Wenn Sie Binärdateien über das Internet verbreiten beachten Sie, dass der Quellcode auf einem Server verfügbar sein muss, der entweder von Ihnen betrieben wird oder von jemandem, mit dem Sie einen ausdrücklichen Vertrag haben.

Vergessen Sie nicht, sich das FSF-HOWTO zur Nutzung der GNU GPL anzusehen. Es ist kurz, verständlich und erklärt Ihnen genau, wie Sie vorgehen müssen, wenn Sie Code unter der GNU GPL veröffentlichen.
