¿Qué es el proyecto GNU?

El Proyecto GNU se fundó en setiembre de 1983 por el estadounidense Richard M. Stallman para crear un sistema operativo completo, similar a Unix, basado en Software Libre. Empezó a trabajar en ello en el enero siguiente.

Stallman puso en marcha la «Free Software Foundation» («Fundación para el Software Libre») en 1985 para enfrentarse con los aspectos organizativos y legales del Proyecto GNU y también para difundir el uso y el conocimiento del Software Libre. A través de la Free Software Foundation se crearon la GNU General Public Licence y la GNU Lesser General Public License (su nombre original era GNU Library Public Licence); con los años, se han establecido como las licencias de Software Libre más usadas globalmente.

El Proyecto GNU consiste en una serie de pequeños subproyectos mantenidos por voluntarios o corporaciones, normalmente con el objetivo de crear o mantener un componente funcional. Estos subproyectos se denominan «GNU Projects» («Proyectos GNU») o «official GNU Projects» («Proyectos oficiales de GNU»).

El nombre del Proyecto GNU proviene del acrónimo recursivo «GNU's Not Unix» («GNU No es Unix»). Originalmente, Unix no era únicamente un tipo de sistema operativo sino que también era un producto, la expresión «GNU» trata de simbolizar que el objetivo del Proyecto GNU es crear un sistema compatible, pero no idéntico a Unix.

El sistema GNU, al igual que otros sistemas operativos Unix, es modular y, hoy en día, especialmente en los sistemas GNU con el núcleo Linux, los llamados sistemas GNU/Linux, que son extensamente utilizados y proporcionan la base para cada una de las llamadas «distribuciones Linux».
