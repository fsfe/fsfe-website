Dieser Überblick hilft Ihnen, die häufigsten Probleme beim Veröffentlichen von Software, die GNU GPL lizensierten Code enthält, zu vermeiden. Die komplette Anleitung finden Sie in der FSF GNU GPL FAQ und im FSF-HOWTO zur Nutzung der GNU GPL.

Diese Dokumente stellen keine Rechtsberatung dar. Konsultieren Sie im Zweifelsfall einen Anwalt.

In aller Kürze:
  1. Stellen Sie sicher, dass der Quellcode der Software für den Anwender zugänglich ist.
  2. Stellen Sie sicher, dass das Urheberrecht der Software bei Ihnen liegt und Sie den Rechten Dritter, deren Software Sie ggf. in ihrem Produkt verwenden, Rechnung tragen.
  3. Stellen Sie sicher, dass Sie die korrekte Version des Quellcodes veröffentlichen. Der mitgelieferte Quellcode muss mit demjenigen identisch sein, aus dem das fertige Programm erstellt wurde.
  4. Stellen Sie sicher, dass Sie den vollständigen Quellcode aller verwendeten (Hilfs-)Programme beigefügt haben, und nicht etwa nur derjenigen Programme oder Programmteile, die selbst schon GNU GPL lizensiert sind.
  5. Stellen Sie sicher, dass das Paket die zum Kompilieren und Installieren notwendigen Skripte enthält.
  6. Falls Sie ein Programm veröffentlichen, das Werkzeuge, die unter der GNU GPL lizensiert wurden, verwendet (z.B. den GCC Compiler), so stellen Sie sicher, dass auch der Quellcode dieser Werkzeuge für den Anwender zugänglich ist.
Vor dem Verteilen der Software:
  1. Überprüfen Sie, ob Ihre Kaufverträge Sie verpflichten, Ihren Lieferanten die Verwendung von GPL-lizensierter Software mitzuteilen.
  2. Überprüfen Sie, ob Sie von Ihren Lieferanten mit allem Notwendigen ausgestattet wurden, um mit der GNU GPL konform zu gehen.
  3. Treffen Sie für den Fall, dass Ihre Lieferanten mit der GPL nicht konform gehen, eine Vereinbarung, aus der hervorgeht, dass diesem Umstand zeitnah abgeholfen wird.
Das Verteilen der Software:
  1. Stellen Sie sicher, dass Ihrem Produkt eine Kopie des Wortlautes der GNU GPL beiliegt.
  2. Stellen Sie sicher, dass entweder der Quellcode der Software oder das schriftliche Angebot, den Quellcode auf einem physischen Medium (etwa einer CD-Rom) auf Verlangen nachzuliefern, Ihrem Produkt beigefügt ist. Für den Versand des Quellcodes auf einem physischen Medium dürfen dem Anfragenden keine höheren Kosten berechnet werden, als die für die Herstellung und den Versand des Mediums anfallen.
  3. Beachten Sie, dass es nicht ausreicht, einen Downloadlink anstelle des Angebotes, den Quellcode auf einem physichen Medium nachzuliefern, anzubieten. Sie können jedoch Ihr Angebot, den Quellcode nachzuliefern, um den Link zu einer Downloadseite ergänzen, wodurch Sie die Häufigkeit des Versendens des Quellcode wahrscheinlich verringern können.
  4. Wenn Sie Software über das Internet verteilen, muss der Quellcode dieser Software auf demselben Server verfügbar sein.

Denken Sie daran, sich das FSF-HOWTO zur Nutzung der GNU GPL anzusehen. Es ist kurz, verständlich und erklärt Ihnen genau, wie Sie vorgehen müssen, wenn Sie Code unter der GNU GPL veröffentlichen wollen.
