Como vendedor de productos basados en GNU GPL, ¿qué es una buena práctica?

  • Recuerde que la licencia le exige a usted poner a disposición el código fuente junto con su producto o incluir una oferta escrita de poner a disposición el código fuente en caso de que se lo pidan .Poner una copia del código fuente en una documentación adjunta en un CD es una forma muy buena de hacerlo.
  • Si usted usa software GPL usted tiene que incluir una copia de GPL con su documentación de licencia y aclarar que su producto contiene software GPL.
  • Si usted ofrece actualizaciones de software o firmware vía Internet, y ellas contienen software GPL, tiene que poner a disposición el código fuente correspondiente a esa actualización.

Esto no es un consejo legal. Si tiene dudas consulte a un consejero legal.

¿Cuáles son los errores típicos?

  • Usar un texto standard boilerplate que declara que todo el software es copyright de su compañía. Esto no es verdad si usted está usando en su producto Software Libre con copyright de terceras partes.
  • Olvidar la oferta escrita de proveer el código fuente.
  • Incluir la versión equivocada del código fuente. El código fuente tiene que ser exctamente la misma version que el código (objeto) ejecutable, que se está vendiendo en ese momento.
  • Olvidar incluir los "scripts usados para controlar la compilación y la instalación del ejecutable"; las erramientas requeridas para compilar e instalar las componentes bajo licencia GPL.
  • Incluir sólo un enlace al texto de licencia GPL en vez de una copia in extenso.
  • Sólo incluir un link para download del código fuente en vez de la oferta escrita de colocar el código fuente en un medio físico de almacenamiento de los usados de costumbre para intercambiar información.
  • En caso de distribución de software haciendo disponible el código públicamente para download: colocando enlaces a un texto de licencia o código fuente en servidores que no sean los suyos. Esto conviene a la sección GPL "3c" (pasando vía oferta escrita) lo cual es solamente válido para distribuciones no-comerciales.

¿Qué debo hacer si nuestra compañia ha violado la licencia?

Fijarlo. Esto funciona exactamente de la misma manera en que usted arreglaría una violación de licencia de software propietario.

Si ud. viola la licencia en GNU/Linux está violando la de innumerables partes, incluyendo compañias como Intel, IBM y HP; todas las tres compañias tienen copyright en porciones significantes del Linux kernel. La violación es una cuestión seria y tiene que ser rectificada.

Como vendedor, ¿dónde puedo obtener ayuda?

Hay una variedad de fuentes de ayuda e información. La primera persona a la que debe llamar es probablemente a su proveedor de componentes de software GPL. Debería consultar además a un abogado. La licencia GPL está escrita para ser lo más clara posible, pero un abogado puede ser útil para asuntos complejos tales como trabajos derivados.

Páginas tales como sourceforge.net pueden ayudar a proveer hosting para versiones de software GPL.

¿Cómo hago feliz a la comunidad?

La mayor contribución que puede hacer es darle acceso al código. Una de las cosas más útiles que puede hacer una compañía es contribuir con los cambios que ella ha hecho. Poner el software en un lugar accesible a todos tal como sourceforge.net es realmente también de ayuda. Algunos vendedores prueban y forman comunidades alrededor de un código y trabajan junto con la comunidad en el desarrollo. Otros simplemente lanzan el release y lo olvidan. La elección está en sus manos.

¿Al usar código GPL me costará mucho dinero?

No. Su deber es proveer o bien el código fuente de las componentes GPL o la oferta escrita de proveerlo por un costo no mayor que la del envío. No hay ninguna obligación con partes terceras, no hay obligación de ofrecer soporte a software que haya sido creado usando sus cambios. La GPL contiene también exclusión de garantía.

Es posible también cubrir los requerimientos de licencia simplemente incluyendo una copia de la licencia GPL en el manual de la aplicación, y créditos apropiados y un archivo zip del código fuente relevante en el CD-ROM de su producto.

La única situación en la que el código GPL le puede costar mucho es si usted se propobe violar la licencia.
