Dicas úteis sobre conformidade para os usuários

Estas dicas podem ajudá-lo a localizar alguns dos problemas comuns do software que contém o código sob a licença GNU GPL. Para obter informações detalhadas e fidedignas sobre a GNU GPL, você deve ler as Perguntas mais frequentes sobre a GNU GPL da FSF.

Isto não é um aconselhamento jurídico. Se você tiver dúvidas consulte um advogado.

Diretrizes básicas

  1. Verifique se uma cópia da licença GNU GPL vem com o software.
  2. Verifique se uma cópia do código-fonte vem com o software ou se houver uma oferta por escrito para fornecer o código-fonte em uma mídia física, como um CD-ROM por não mais do que o custo de produção e transporte.
  3. Verifique se o distribuidor não reclama um copyright exclusivo sobre o software, se o software também contém código criado por outros.

Diretrizes mais detalhadas

  1. Verifique se a versão correta do código-fonte está disponível. O código fonte tem que ser exatamente da mesma versão que o código executável (objeto) fornecido.
  2. Verifique se o código fonte inclui a fonte de quaisquer trabalhos derivados do código GNU GPL, e não apenas dos componentes GNU GPL de terceiros.
  3. Verifique se o código fonte inclui os scripts utilizados para controlar a compilação e instalação.
  4. Se o conjunto de ferramentas é lançado e contém ferramentas sob a GNU GPL (como o compilador GCC), verificar se o código fonte dessas ferramentas também está disponível.

Se você está preocupado de que alguns softwares que você tem recebido não estão cumprindo os termos da licença GNU GPL pode rever as Perguntas mais frequentes sobre a GNU GPL da FSF e o HOW-TO para usar a GNU GPL da FSF. Se você suspeita que houve uma violação da licença GNU GPL pode contatar a FTF e nós vamos ajudá-lo a resolver o problema.
